# 命名规则

## 文件和文件夹

1. 字母全部小写，单词间用小短线-连接
2. 如果一个文件夹下有多个同类型文件，则文件名应该是复数

```javascript
utils / components / auto - complete / data - picker.tsx;
data - picker.scss;
logo - knowledge.png;
knowledge - intro.mp4;
```

## React 中命名

1. 组件名首字母大写（PascalCase）

```tsx
function App() {
  return (
    <div>
      <Button />
    </div>
  );
}

function Button() {
  return <button>Click me</button>;
}
```

2. 样式属性名使用驼峰命名法（camelCase）

```tsx
const styles = {
  button: {
    backgroundColor: "blue",
    color: "white",
    padding: "10px 20px",
  },
};
```

## 使用具名导出而不是默认导出的方式来处理工具函数

**✅ Good:**

```tsx
export function getDate() {
  // ...
}

export const getDate = () => {
  // ...
};
```

**❌ Bad:**

```tsx
function getDate() {
  // ...
}

export default getDate;
```

### 常量命名 (UPPER_CASE)

```javascript
const MAX_VALUE = 100;
```

### 变量命名 (camelCase)

```javascript
let thisIsMyName;

let goodID;

let reportURL;

let AndroidVersion;

let iOSVersion;
```

<Callout type="default">
  变量名不应少于三个字符(循环中使用的迭代变量 i,j,k
  等除外)，过短的名字不能体现其代表等意义，将使代码变得不可读 <br />
  使用的 ID 的地方一定全大写 <br />
  使用的 URL 的地方一定全大写 <br />
  涉及 Android 的，一律大写第一个字母 <br />
  涉及 iOS 的，一律小写第一个，大写后两个字母
</Callout>

### 函数命名 (camelCase)

```javascript
function myFunction() {
  // ...
}

function handleShowCategory() {
  // ...
}

function onClickTitle() {
  // ...
}

function fetchData() {
  // ...
}

function Person(name) {
  this.name = name;
}
```

<Callout type="default">
  函数的名称应尽可能具有自解释性。处理用户交互的函数名称以 “handle” 或者 “on”
  开头 <br />
  函数名应使用动词或动词短语，如 `fetchData` 或 `calculateTotal` <br />
  构造函数大写第一个字母
</Callout>

## 布尔值命名

```javascript
let isDone = true;
let hasError = false;
let canEdit = true;
let shouldShow = false;
function isUserLoggedIn() {}
```

<Callout type="default">
  在布尔值前加上 `is`, `has`, `can` 或者 `should` (e.g. `isLoading`, `hasTitle`
  或者 `shouldShowImage`)
</Callout>

## 使用 `use` 前缀来命名 `React Hook` 函数

```tsx
function useFetchData() {
  // ...
}

const useFetchData = () => {
  // ...
};
```

      <FileTree.File name="getWindowHeight.js" />

  </FileTree.Folder>

`viewport.js` 如下所示:

```javascript
// As a function declaration
export function getWindowHeight() {
  // ...
}

export function getScrollPosition() {
  // ...
}

// Or as an arrow function
export const getWindowHeight = () => {
  // ...
};

export const getScrollPosition = () => {
  // ...
};
```

<Callout type="default">
  按照包含的函数的主题来命名实用工具文件，这样多个函数可以相邻存放而无需为每个函数创建一个新文件。这使得实用工具更容易被找到
</Callout>
